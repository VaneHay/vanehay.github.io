git常用命令

Nov 27, 2018

Git 全局设置
git config –global user.name “Administrator”
git config –global user.email “vanehay@163.com“

局部配置
$ git config user.name “小甲鱼” //给自己起个用户名
$ git config user.email “VaneHay@163.com” //填写自己的邮箱

创建新版本库

git clone git@git.sxvol.com:root/test.git
cd test
touch README.md
git add README.md
git commit -m “add README”
git push -u origin master

已存在的文件夹

cd existing_folder
git init
git remote add origin git@git.sxvol.com:root/test.git
git add .
git commit
git push -u origin master

已存在的 Git 版本库

cd existing_repo
git remote add origin git@git.sxvol.com:root/test.git
git push -u origin –all
git push -u origin –tags

===============================================================
修改从仓库拉取的文件后进行提交操作

git add —-update

git commit -m “add 说明”

git push -u origin master

===============================================================
git拉去仓库最新版本到本地
git fetch origin master

拉取最新版并且于本地的合并
git pull origin master

git log

git log –pretty=oneline

增加使用说明文档

在Git中，用HEAD表示当前版本，也就是最新的提交3628164…882e1e0（注意我的提交ID和你的肯定不一样），上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100

git reset –hard HEAD^ 回滚一次

git rm a.txt ，误将a.txt删除后找回方法：

  1. git log 找到离没删文件前最近的commit id
  2. 将操作过的其它文件转移
  3. git reset –hard “commit id”
